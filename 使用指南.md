# NovelOS 使用指南

> 一句话介绍：NovelOS 是一个**长篇小说工程系统**，帮你用AI写出百万字级别的网文，而且不会"忘记"前面的剧情。

## 快速开始

### 第一步：安装依赖

```bash
cd NovelOS
pip install pydantic pyyaml click pytest
```

### 第二步：初始化项目

在Trae中，直接告诉AI：

```
帮我创建一个新的末日题材小说项目，名字叫"末日重启"
```

AI会自动：
- 创建项目目录结构
- 生成故事圣经模板
- 创建大纲文件

### 第三步：开始写作

```
帮我生成第一章，主角陈萧在废墟中醒来
```

## 核心概念

### 1. 故事圣经 (Story Bible)

**是什么**：记录小说中所有重要信息的"数据库"

**包含什么**：
- 角色：名字、性格、能力、当前状态
- 世界观：设定、规则、势力
- 时间线：事件发生顺序
- 伏笔：埋下的线索和解决状态
- 关系：角色之间的关系

**为什么重要**：
> 写到第200章时，AI还能记得第3章埋的伏笔，就是因为故事圣经记录了一切。

### 2. 章节上下文 (Context)

**是什么**：生成新章节时，AI需要知道的"背景信息"

**包含什么**：
- 前几章的摘要
- 相关角色的当前状态
- 未解决的伏笔
- 相关的世界设定

**为什么重要**：
> 没有上下文，AI就会"忘记"前面发生了什么，写出矛盾的内容。

### 3. 连贯性检查 (Coherence Check)

**是什么**：自动检测章节之间的问题

**检测什么**：
- 重复场景（两章内容太像）
- 角色位置跳跃（上一章在A地，这章突然在B地）
- 时间线错误
- 角色状态矛盾

**为什么重要**：
> AI生成的内容经常有重复，连贯性检查帮你自动发现问题。

## 在Trae中使用

### 方式一：自然语言交互

直接用中文告诉AI你想做什么：

```
# 创建项目
帮我创建一个无限流小说项目

# 规划章节
帮我规划前10章的大纲

# 生成章节
帮我生成第5章，主角进入第一个副本

# 检查连贯性
检查第5章和前面的章节有没有矛盾

# 查看故事状态
告诉我主角现在在哪里，有什么能力
```

### 方式二：使用CLI命令

```bash
# 查看项目状态
python -m app.cli status

# 检查章节连贯性
python -m app.cli coherence check --chapter 5

# 检查所有章节
python -m app.cli coherence check --all

# 生成连贯性报告
python -m app.cli coherence report --chapter 5
```

## 工作流程

### 标准流程

```
1. 规划 → 2. 生成 → 3. 检查 → 4. 评审 → 5. 修订 → 6. 更新
```

#### 1. 规划

```
帮我规划第10章，主角要遇到一个新角色
```

AI会：
- 查看故事圣经
- 确认当前剧情进度
- 生成章节规划

#### 2. 生成

```
根据规划生成第10章
```

AI会：
- 构建上下文
- 生成章节内容
- 自动检查连贯性

#### 3. 检查

```
检查第10章有没有问题
```

AI会检测：
- 重复场景
- 角色状态矛盾
- 时间线错误

#### 4. 评审

```
评审第10章的质量
```

AI会评估：
- 剧情是否合理
- 节奏是否合适
- 有没有AI味

#### 5. 修订

```
根据评审意见修改第10章
```

AI会：
- 修改有问题的部分
- 保持其他内容不变

#### 6. 更新

```
更新故事圣经
```

AI会自动提取：
- 角色状态变化
- 新出现的能力
- 新的关系
- 新的伏笔

## 常见问题

### Q: AI为什么会"忘记"前面的剧情？

**A**: 因为AI的上下文窗口有限，无法记住所有内容。NovelOS通过**故事圣经**和**章节摘要**解决这个问题，每次生成新章节时，只加载相关的信息。

### Q: 如何避免章节内容重复？

**A**: NovelOS的**连贯性检查服务**会自动检测重复场景。生成新章节后，运行：
```
检查第X章的连贯性
```

### Q: 如何让AI记住角色的能力变化？

**A**: 每章生成后，NovelOS会自动提取状态变化并更新故事圣经。你也可以手动查看：
```
告诉我陈萧现在有什么能力
```

### Q: 支持什么题材？

**A**: 支持所有网文题材：
- 末日/末世
- 无限流
- 玄幻/修仙
- 都市异能
- 悬疑推理
- ...

### Q: 能写多长？

**A**: 理论上没有限制。NovelOS设计目标是支持**百万字级别**的长篇小说。
